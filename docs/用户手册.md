## 1. 产品简介

**COCOVisualizer** 是一款面向计算机视觉工程师的本地 COCO 数据集管理与分析工具，专为以下场景设计：

- **数据审查**：批量浏览图片与标注框，快速发现标注错误
- **图片级打标**：为每张图片附加自定义分类标签（误检、漏检、高置信度等），无需改动标注格式
- **数据分析**：类别分布、框尺度、中心点、密度等多维统计图表
- **AI 辅助分析**：内置 Agent，可用自然语言提问并执行 Python 代码分析数据
- **实验导出**：按分类或按 train/val 比例导出子集 ZIP

### 支持的数据格式

| 格式 | 说明 |
|---|---|
| 标准 COCO JSON | `images` / `annotations` / `categories` 三段式 |
| 含预测结果的 COCO | 标注中含 `score` 字段，支持置信度过滤 |
| 多目录合并 | 约定每个子目录含 `_annotations.coco.json`，自动合并 |
| 图片级分类扩展 | 文件中含 `image_category_definitions` 字段时，自动加载分类定义 |

## 2. 安装与启动

### 2.1 Python 源码模式（推荐开发/调试）

```bash
# 克隆仓库
git clone <repo-url>
cd COCOVisualizer

# 安装依赖
pip install -r requirements.txt

# 启动（默认端口 6010）
python app.py

# 指定端口
python app.py --port 8080

# 启动后不自动打开浏览器
python app.py --no-browser
```

服务启动后自动在浏览器打开 `http://127.0.0.1:6010`。

### 2.2 macOS 应用包（双击启动）

将 `COCO-Visualizer.app` 拖入「应用程序」文件夹，双击即可启动。
首次启动 macOS 会提示「无法验证开发者」，前往 **系统设置 → 隐私与安全性 → 仍要打开** 放行即可。

### 2.3 前端开发模式（Vite Dev Server）

```bash
# 终端 1：启动后端
python app.py --port 6010

# 终端 2：启动前端热重载
cd frontend
npm install
npm run dev          # 默认 http://127.0.0.1:5173
```

### 2.4 生产打包（前端 + PyInstaller）

```bash
# 先构建前端
cd frontend && npm run build && cd ..

# 再打包应用
./scripts/build_mac_app.sh
# 输出：dist/COCO-Visualizer.app
```

## 4. 加载数据集

### 4.1 加载预测结果（GT 对比核心功能）

> ⚡ **这是最常用的核心场景**：将模型推理输出的 COCO 文件与 GT 标注并排对比，分析误检/漏检。

#### 文件命名约定（唯一要求）

将预测 COCO 文件放到与 GT 文件**同一目录**，按以下规则命名：

```
_annotations.<模型名>.pred.coco.json
```

| 文件名示例 | 说明 |
|---|---|
| `_annotations.yolov8.pred.coco.json` | YOLOv8 推理结果，模型标签为 `yolov8` |
| `_annotations.rtdetr.pred.coco.json` | RT-DETR 推理结果，模型标签为 `rtdetr` |
| `_annotations.v1.pred.coco.json` | 任意自定义名称均可 |

目录结构示例：

```
/data/my_dataset/
├── _annotations.coco.json               ← GT（主文件，正常加载）
├── _annotations.yolov8.pred.coco.json   ← 预测文件 1（自动识别）
└── _annotations.rtdetr.pred.coco.json   ← 预测文件 2（自动识别）
```

> **无需任何额外操作**：只要命名符合约定，重新加载数据集后预测框自动挂载。查看器左侧面板随即出现预测结果；图库卡片显示预测框数；Agent 沙箱变量 `pred_annotations` 可直接访问。

#### 预测文件 COCO 格式

标准 COCO JSON，annotations 中需加 `score` 字段（置信度 0–1）：

```json
{
  "images": [
    {"id": 1, "file_name": "images/0001.jpg"}
  ],
  "annotations": [
    {"id": 1, "image_id": 1, "category_id": 1,
     "bbox": [100, 200, 300, 150], "score": 0.92},
    {"id": 2, "image_id": 1, "category_id": 1,
     "bbox": [400, 300, 120, 80],  "score": 0.45}
  ],
  "categories": [{"id": 1, "name": "defect"}]
}
```

**注意事项：**
- `score` 字段：取值 0–1，工具据此提供置信度阈值滑块；无 score 时仍可显示但无法过滤
- `file_name` 匹配：按 basename 匹配 GT 图片（忽略目录前缀），例如 GT 中的 `images/0001.jpg` 与预测中的 `0001.jpg` 可自动对应
- 多模型同时对比：放多个 `*.pred.coco.json` 文件，查看器左侧会出现模型选择下拉框

#### 加载后可用的对比功能

| 功能 | 说明 |
|---|---|
| 预测框显示 | 查看器左侧面板，独立置信度阈值滑块 |
| GT vs 预测对比 | 点击工具栏「⊞ 对比」，双栏同步缩放 |
| IoU 联动高亮 | 悬停一侧框项，另一侧匹配框高亮 |
| 预测框转 GT | 点击「+GT」单条接受，或「全部接受」批量导入 |
| AP/mAP 评估 | Agent 中调用 `compute_ap()` 或直接提问 |
| 代码分析 | `img['pred_annotations']` 访问每张图的预测框列表 |

### 4.2 多目录合并扫描（推荐）

适合大量子目录、约定命名的数据集（Magic-Fox 导出格式）。

**操作步骤：**

1. 切换至「多目录合并扫描」标签
2. 在输入框粘贴根目录路径（支持多个，用 `;` 分隔或每行一个）
   - macOS 示例：`/Users/name/datasets/project_v2`
   - Windows 示例：`D:\data\project_v2`
3. 也可直接**拖拽文件夹**到输入区，路径自动填入
4. 点击「**开始扫描**」
5. 扫描结果显示所有发现的 `_annotations.coco.json` 子目录，勾选需要合并的条目
6. 填写合并后数据集名称（可选）
7. 点击「**合并加载**」

> **约定**：工具自动查找 `<根目录>/**/_annotations.coco.json`，同目录内所有 `*.pred.coco.json` 也会一并自动挂载。

### 4.3 单文件精准加载

适合指定具体文件路径的场景。

**操作步骤：**

1. 切换至「单文件精准加载」标签
2. 填写 GT COCO JSON 路径（必填）
3. 填写图片目录（可选，留空时从 JSON 同目录推断）
4. 填写数据集名称（可选，用于区分多次加载记录）
5. 点击「**加载**」

> 预测文件只需与 GT 文件在同一目录且命名符合约定，**无需单独指定路径**。

### 4.4 最近记录

加载页底部显示最近 6 次加载记录，点击可一键重新加载。

### 4.5 服务端路径浏览

点击「浏览目录...」按钮，可在**运行后端的机器**上浏览目录结构，
适合远程 SSH 部署场景（本机运行工具但数据在服务器上）。

## 5. 图库浏览

数据集加载完成后自动跳转到图库页。

### 5.1 工具栏功能

```
┌────────────────────────────────────────────────────────────────┐
│ [筛选栏] 类别 · 时间 · SN号 · 位置    [排序] [保存] [导出] ... │
│ ──────────────────────────────────────────────────────────────  │
│ [全选] □  代码筛选  排序▾  ⚠无GT N                             │
│ ──────────────────────────────────────────────────────────────  │
│ [图片宫格区域]                              [右侧统计面板]      │
└────────────────────────────────────────────────────────────────┘
```

### 5.2 筛选图片

#### 按 GT 标注类别筛选

顶部下拉框「全部标注类别」→ 选择具体类别，图库只显示含该类别标注的图片。

#### 按图片分类筛选

右侧统计面板 → 点击分类标签（如「未分类」「误检」），图库高亮/过滤对应图片。

#### 按元数据筛选

点击「**应用筛选**」展开时间、SN号、位置等元数据筛选栏（需数据集包含相应字段）。

#### 按文件名搜索

顶部「搜索文件名...」输入框，支持模糊匹配。

#### Agent 代码筛选

点击「**代码筛选**」按钮，在编辑器中编写 Python 表达式，最后一行返回 `image_id` 列表即可作为筛选结果。

示例：
```python
# 筛选 GT 框数量超过 5 个的图片
[img['image_id'] for img in images if len(img.get('annotations', [])) > 5]
```

### 5.3 图片批量操作

1. 勾选图片（顶部「全选当页」快速全选）
2. 对选中图片进行：
   - **批量设置分类**：右侧面板点击分类按钮，应用到所有选中图片
   - **批量清空标注**：工具栏「🗑 清空 GT」（不可撤销，操作前确认）

### 5.4 图片排序

工具栏排序下拉，支持：
- 默认顺序（按 image_id）
- 文件名 A→Z / Z→A
- GT 框数量 多→少 / 少→多
- 文件大小 大→小 / 小→大
- 修改时间 新→旧 / 旧→新

### 5.5 右侧统计面板

- **图片级分类统计**：每个分类的图片数量与占比，点击可过滤
- **GT 标注类别统计**：每个 GT 类别的框数量，点击可过滤

### 5.6 保存图片分类与备注

系统会自动保存（防抖 1 秒），状态栏显示「已自动保存」。
也可手动点击「💾 保存」，输入版本说明后写入 COCO 文件并生成版本快照。

## 7. 数据分析（EDA）

点击侧边栏「📊」进入数据分析页，需已加载数据集。

### 7.1 可用图表

| 分析维度 | 图表 |
|---|---|
| **类别分布** | 饼图 + 柱状图 |

## 8. AI 智能助手

点击侧边栏「💬」进入 Chat 页，可在不加载数据集的情况下单独使用。

### 8.1 配置 LLM

首次使用需在 `static/config.json` 配置 API：

```json
{
  "llm": {
    "apiUrl": "https://api.openai.com/v1/chat/completions",
    "apiKey": "sk-xxxxx",
    "model": "gpt-4o",
    "maxTokens": 4000
  }
}
```

或使用任何兼容 OpenAI 接口的服务（Ollama、DeepSeek、通义千问等）。

### 8.2 对话功能

- **自然语言提问**：直接问「当前数据集有多少张图片？」「各类别框的平均面积是多少？」
- **代码生成与执行**：Agent 自动编写 Python 代码并在后端沙箱执行，返回结果
- **图表生成**：代码中调用内置 `show_chart(...)` 函数，自动渲染 Plotly 图表
- **数据表格**：代码中调用 `show_table(...)` 函数，渲染交互式表格
- **图库联动**：代码中调用 `filter_gallery(image_ids)` 函数，过滤图库显示

### 8.3 内置可用函数

在 Agent 生成的代码中，以下函数和变量自动注入：

| 名称 | 类型 | 说明 |
|---|---|---|
| `images` | `list[dict]` | 当前加载的图片列表（含 annotations） |
| `dataset_id` | `str` | 当前数据集 ID |
| `show_chart(data)` | 函数 | 在 Chat 气泡内渲染 Plotly 图表 |
| `show_table(data)` | 函数 | 渲染数据表格 |
| `filter_gallery(ids)` | 函数 | 用 image_id 列表过滤图库 |
| `navigate(page)` | 函数 | 切换页面（gallery/eda/load）|
| `export_csv(data, name)` | 函数 | 导出 CSV 文件供下载 |
| `export_json(data, name)` | 函数 | 导出 JSON 文件供下载 |

### 8.4 多会话管理

左侧会话列表支持：
- 新建对话（`+` 按钮）
- 切换历史会话
- 删除会话（hover 显示删除按钮）
- 会话持久化（刷新页面不丢失）

### 8.5 附件上传

输入框旁「📎」按钮，可上传图片作为视觉上下文发送给多模态模型。

### 8.6 代码手动运行

消息气泡中的代码块右上角「▶ 运行」按钮，可重新执行某段代码。

### 8.7 自定义 Agent 模块

在「⚙️ 设置」→「Agent 模块」中，上传自定义 Python 文件，为 Agent 注册专用函数（如调用内部 API、计算自定义指标等）。

## 9. 导出功能

在图库页点击工具栏「📤 导出」按钮，弹出导出弹窗。

### 9.1 按分类导出

**用途**：把不同图片分类（如「误检」「漏检」）分别导出为独立 COCO ZIP 文件。

**步骤**：
1. 切换至「按分类」标签
2. 勾选要导出的分类
3. 点击「导出」
4. 浏览器自动下载 ZIP，内含：
   - 每个分类一个子目录（`<分类名>/images/` + `_annotations.coco.json`）

### 9.2 实验数据集（train / valid）

**用途**：将当前 COCO 中的图片按比例划分为训练集和验证集，打包为实验 ZIP。

**参数说明**：

| 参数 | 说明 | 默认值 |
|---|---|---|
| 训练集比例 | 0.0–1.0，剩余为验证集 | 0.8 |
| 随机种子 | 固定后划分结果可复现 | 42 |
| ZIP 文件名 | 输出文件名（加 `.zip`）| dataset_split.zip |
| 服务端保存路径 | 可选，留空则触发浏览器下载 | 空 |
| 丢弃重名图片 | 同文件名只保留 image_id 最小的一张 | 开启 |

**ZIP 结构**：
```
dataset_split.zip
├── train/
│   ├── images/
│   └── _annotations.coco.json
└── valid/
    ├── images/
    └── _annotations.coco.json
```

> **注意**：实验导出使用当前已加载的全部图片，图库中的分类筛选不影响导出范围。

## 10. 版本管理

### 10.1 版本创建

每次点击「💾 保存」并填写版本说明，系统在 COCO 文件同目录的 `.coco_visualizer/` 隐藏文件夹内创建一条版本快照。

### 10.2 查看版本记录

图库工具栏「📋 版本」按钮 → 弹出版本列表，显示：
- 版本创建时间
- 版本说明
- 标记「最新」的当前状态

### 10.3 回滚版本

在版本列表中点击「回滚到此版本」→ 确认后 COCO 文件恢复为该版本内容，图库同步刷新。

> **提示**：回滚操作本身也会产生一条新版本记录，便于反悔再次回滚。

### 10.4 版本文件共享

将 `.coco_visualizer/` 目录连同 COCO 文件一起打包，在其他机器打开同一 COCO 文件时版本记录即可还原。

**全局默认分类**：当数据集中没有内嵌分类定义时使用。

## 13. 常见问题

### Q1：页面显示「脚本加载失败」

**原因**：前端资源（React、Babel 或 Vendor JS）加载失败。

**解决**：
1. 确认 `python app.py` 已正常启动且无报错
2. 检查 `static/vendor/` 目录是否包含 `plotly.min.js` 等文件
3. 如在离线环境，运行 `python packaging/fetch_vendor_js.py` 下载依赖

---

### Q2：加载 COCO 文件后图片显示为空白或 404

**原因**：图片路径无法解析。

**解决**：
- 在「单文件精准加载」中手动指定「图片目录」字段
- 确认图片文件与 COCO JSON 在同一机器上可访问
- 若 `file_name` 字段含绝对路径（如 `/data/xxx.jpg`），确保该路径在服务器上存在

---

### Q3：自动保存频繁写入，文件被占用

**解决**：
- 自动保存使用 `skip_version=true`，不产生版本快照，仅更新分类和备注字段
- 如不需要自动保存，可在代码中调大防抖时间（`backend/services/annotation_service.py`）

---

### Q4：AI Chat 报错「请求失败」

**解决**：
1. 检查 `static/config.json` 中 `llm.apiKey` 是否填写
2. 检查 `llm.apiUrl` 是否可访问（如使用系统代理注意本地环回地址的代理设置）
3. 查看终端后端日志，确认是 API 错误还是网络错误

---

### Q5：扫描目录后找不到数据集

**解决**：
- 工具只扫描文件名为 `_annotations.coco.json` 的文件
- 确认子目录中文件名符合约定，或切换「单文件精准加载」手动指定路径

---

### Q6：如何在远程服务器上使用

```bash
# 服务器端启动（监听所有地址）
python app.py --port 6010

# 本机通过 SSH 隧道访问
ssh -L 6010:127.0.0.1:6010 user@server

# 浏览器访问
http://127.0.0.1:6010
```

---

### Q7：多人能否同时访问同一个实例

Flask 本身支持多客户端同时连接。但注意：

- 图片分类、备注修改在后端共享同一份内存状态
- 自动保存写入同一 COCO 文件，多人同时修改不同图片不会冲突
- **不建议**多人同时对同一图片进行修改操作
